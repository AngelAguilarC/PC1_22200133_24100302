PRACTICA CALIFICADA 01 - PREGUNTAS DE REFLEXION
================================================

a. Dificultades durante el scaffolding y como las solucionaste:

La principal dificultad fue que al correr Scaffold-DbContext la primera vez no encontraba
el proveedor de SQL Server porque no tenia instalado el paquete Microsoft.EntityFrameworkCore.SqlServer
en el proyecto correcto. Lo solucione instalando el paquete en la biblioteca de clases donde
iba a generar las entidades. Tambien tuve que asegurarme de que el proyecto de herramientas
(Microsoft.EntityFrameworkCore.Tools) estuviera instalado en el proyecto de inicio.
Otro problema fue que la cadena de conexion estaba mal escrita al principio,
lo corregi revisando el formato correcto para localdb.

b. Diferencias entre controlador autogenerado y con repository. Enfoque preferido:

El controlador autogenerado (TipoServicio) es mas simple porque accede directamente al DbContext,
lo que lo hace rapido de implementar.
El controlador con repository (OrdenServicio) esta mejor organizado porque separa responsabilidades:
el controlador solo maneja HTTP, el service maneja la logica del negocio y el repository maneja
el acceso a datos. Aunque requiere mas archivos, es mas facil de mantener y de hacer pruebas unitarias.
Prefiero el patron repository para proyectos reales, aunque para algo rapido el directo esta bien.

c. Buenas practicas implementadas y por que:

- Uso de DTOs (CreateOrdenServicioDto, OrdenServicioDto): evita exponer directamente las entidades
  del dominio y permite controlar que datos se reciben y se envian.
- Inyeccion de dependencias: registre los servicios y repositorios en Program.cs usando
  AddScoped, lo que permite cambiar implementaciones sin tocar el controlador.
- Separacion en capas (Core / Infrastructure): el Core no depende de implementaciones concretas,
  solo de interfaces. Esto sigue el principio de inversion de dependencias.
- Uso de async/await en todos los metodos del repositorio y servicio para no bloquear el hilo
  principal y mejorar el rendimiento bajo carga.

Si, use como apoyo durante la practica. Principalmente para:
- Generar la estructura inicial del proyecto, apoyarme en la generación de la BD.
