COMO HACER FUNCIONAR EL PROYECTO
=================================

PASO 1 - Crear la base de datos en SQL Server
----------------------------------------------
1. Abre SQL Server Management Studio (SSMS)
2. Conectate a (localdb)\mssqllocaldb
3. Abre un New Query
4. Abre el archivo BaseDeDatos.sql que esta en la carpeta del Web API y pega su contenido
5. Presiona F5 para ejecutar
6. Verifica que aparezca TallerMecanicoDB en el explorador de objetos


PASO 2 - Abrir la solucion en Visual Studio
--------------------------------------------
1. Abre Visual Studio 2022
2. Ve a File > Open > Project/Solution
3. Navega a PC1_22200133_24100302 en el Escritorio y abre PC1_22200133_24100302.slnx
4. Deberias ver dos proyectos en el Solution Explorer:
   - PC1_22200133_24100302 (Web API)
   - TallerMecanico.Library (biblioteca)


PASO 3 - Restaurar paquetes NuGet
----------------------------------
En Visual Studio:
- Click derecho sobre la solucion (no el proyecto) > Restore NuGet Packages

O en la Package Manager Console (Tools > NuGet Package Manager > Package Manager Console):
   dotnet restore


PASO 4 - Verificar la cadena de conexion
-----------------------------------------
Abre PC1_22200133_24100302/appsettings.json y confirma que la connection string apunte a tu servidor.
Por defecto esta:

   "TallerMecanicoDB": "Server=(localdb)\\mssqllocaldb;Database=TallerMecanicoDB;Trusted_Connection=True;TrustServerCertificate=True;"

NOTA: Si usas SQL Server Express en lugar de localdb, cambia (localdb)\\mssqllocaldb por .\SQLEXPRESS


PASO 5 - Compilar la solucion
------------------------------
Presiona Ctrl+Shift+B o ve a Build > Build Solution.
Deberia compilar sin errores. Si hay errores de paquetes no encontrados, repite el Paso 3.


PASO 6 - Ejecutar la API
-------------------------
1. Asegurate de que el proyecto de inicio sea PC1_22200133_24100302
   (click derecho sobre el proyecto > Set as Startup Project)
2. Presiona F5 o el boton verde de Run
3. La API corre en https://localhost:7272
4. Puedes ir a https://localhost:7272/openapi/v1.json para verificar que los endpoints esten disponibles


PASO 7 - Probar con Postman
----------------------------
1. Abre Postman
2. Click en Import
3. Selecciona el archivo: PC1_22200133_24100302/Pregunta6/TallerMecanico.postman_collection.json
4. La coleccion aparece con dos carpetas: TipoServicio y OrdenServicio

Orden de prueba recomendado:

Primero inserta datos de apoyo en SSMS para poder probar OrdenServicio:

   USE TallerMecanicoDB;

   INSERT INTO Cliente (Paterno, Materno, Nombres, Correo, Telefono)
   VALUES ('Garcia', 'Lopez', 'Juan', 'juan@mail.com', '999888777');

   INSERT INTO Vehiculo (Placa, Marca, Modelo, Anio, ClienteId)
   VALUES ('ABC-123', 'Toyota', 'Corolla', 2020, 1);

Luego en Postman:
1. POST TipoServicio  - crea un tipo de servicio
2. POST OrdenServicio - crea una orden
3. GET todos / por ID
4. PUT para actualizar
5. DELETE para eliminar


ENDPOINTS DISPONIBLES
======================

TipoServicio:
   GET    /api/TipoServicio          Lista todos
   GET    /api/TipoServicio/{id}     Obtiene uno por ID
   POST   /api/TipoServicio          Crea uno nuevo
   PUT    /api/TipoServicio/{id}     Actualiza
   DELETE /api/TipoServicio/{id}     Elimina

OrdenServicio:
   GET    /api/OrdenServicio         Lista todos
   GET    /api/OrdenServicio/{id}    Obtiene uno por ID
   POST   /api/OrdenServicio         Crea uno nuevo
   PUT    /api/OrdenServicio/{id}    Actualiza
   DELETE /api/OrdenServicio/{id}    Elimina
